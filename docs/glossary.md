# Glossary

## Terms

**Base**: The placeholder file that keeps the repository non-empty.

**Bootstrap**: The process of setting up a development environment, handled by `scripts/bootstrap.sh`.

**Changelog**: A record of notable changes, maintained in `CHANGELOG.md`.

**Code Owner**: The person responsible for reviewing changes, defined in `CODEOWNERS`.

**Mission**: The repository's purpose and goals, documented in `docs/mission.md`.

**Roadmap**: Planned future work, tracked in `docs/roadmap.md`.
